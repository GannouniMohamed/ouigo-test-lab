# Ouigo Test Lab — Plan de Polish UI

**But :** rapprocher fidèlement l'UI de l'app du design Claude (écrans macOS + Windows 11), sans casser les tests existants (69 unit + 3 E2E) ni la CI.

**Source de vérité :** `docs/design-claude/` (support.js + .dc.html) et la page « print » (chrome par OS via prop `os="mac"|"win"`).

**Méthode :** 1 ticket = 1 branche `feat/POL-xx` = 1 PR, CI verte (macOS+Linux+Windows) avant merge. Les tests renderer existants (texte/rôles) doivent rester verts ; on ajoute des tests seulement quand un comportement change (contrôles fenêtre, titre par page). Vérification visuelle par screenshot de l'app à la fin.

## POL-01 — Barre de titre frameless par OS (HEADLINE)
**Fichiers :** `src/main/index.ts` (frameless par OS), `src/main/ipc/register.ts` (+IPC `window:minimize/maximize/close`), `src/preload/index.ts` + `src/renderer/api.d.ts` (+`platform`, +`windowControls`), `src/renderer/components/TitleBar.tsx` (nouveau), `src/renderer/App.tsx` (intègre TitleBar + titre par route), test `tests/renderer/titleBar.test.tsx`.
- macOS : `titleBarStyle: 'hiddenInset'`, `trafficLightPosition {x:18,y:18}`, frame natif conservé. Barre custom 42px, titre centré « OuiTest — <page> » 12.5px `#94A3B8`, padding gauche pour les feux, `-webkit-app-region: drag`.
- Windows/Linux : `frame:false`. Barre 40px : icône+titre à gauche, contrôles Fluent à droite (min/max/close, 46px, close hover `#E81123`), via IPC.
- Titre par route : Scénarios→« Hub de tests E2E », Exéc.→« Exécution en cours », Rapports→« Rapports », /run→« Exécution en cours », /report→« Rapport d'exécution », /scenarios/new→« Nouveau scénario ».
- Zone draggable + boutons `-webkit-app-region: no-drag`.
- Test : sur plateforme win (mock `window.api.platform="win32"`) les 3 boutons appellent les bons IPC ; sur mac, pas de boutons custom (feux natifs).
